## OOP guide

#### Core Principles

1. **Encapsulation**: Use private fields (`#field`) to hide internal implementation details
2. **Inheritance**: Create specialized components by extending base classes
3. **Polymorphism**: Implement consistent interfaces that can be used interchangeably
4. **Abstraction**: Hide complex implementation behind simple, clean APIs

#### Approach

- Use components approach
- For components creation follow the ../components/components-creation.md
